# AI 모델 활용 명세 — 붙임2

FiTuna 자체는 AI 모델을 내장/배포하지 않는 **오케스트레이션 도구**입니다.
사용자가 실행 시점에 지정하는 LLM(예: Llama-3-8B-Instruct, Qwen2.5-7B 등)을
양자화/벤치마크할 뿐, 특정 모델을 저장소에 포함하거나 고정하지 않습니다.
아래 표는 사용자가 모델을 선택할 때마다 채워야 하는 템플릿입니다.

| 항목 | 내용 |
|------|------|
| 활용 유형 | 1. 외부 모델 그대로 활용 (FiTuna는 파인튜닝/재학습을 수행하지 않고, 사용자가 지정한 기존 공개 모델의 GGUF 변환·양자화·벤치마크만 수행) |
| 기반 모델명 | (실행 시점에 사용자가 `--model`로 지정한 모델명을 기재. 예: Llama-3-8B-Instruct, Qwen2.5-7B-Instruct) |
| 기반 모델 라이선스 | (해당 모델의 공식 라이선스를 기재. 예: Llama 3 Community License, Apache-2.0 등 — 모델마다 상이하므로 사용 전 반드시 확인) |
| 가중치 출처 URL | (예: https://huggingface.co/meta-llama/Meta-Llama-3-8B-Instruct — 실행 시점에 사용자가 실제 다운로드한 URL 기재) |
| FiTuna 소스 코드 라이선스 | MIT (본 저장소 `LICENSE` 참고) |
| FiTuna의 모델 관여 범위 | 가중치를 수정하지 않음. `llama-quantize`로 정밀도만 낮추고(GGUF 양자화), `llama-bench`/`llama-perplexity`로 처리량과 품질손실을 측정. 학습/파인튜닝/증류 없음 |
| 상용 AI 보조 도구 활용 범위 | (본 프로젝트 개발 과정에서 사용한 AI 코딩 보조 도구가 있다면 이름과 활용 범위를 기재. 예: "코드 스캐폴딩 및 리팩터링 보조에 Claude Code 사용, 핵심 로직 설계와 검증은 개발자가 직접 수행") |

## 작성 안내

- 이 문서는 대회 제출 시 실제로 사용/시연한 모델 1개 이상에 대해 위 표를
  구체적으로 채워 제출해야 합니다 (플레이스홀더 상태로 제출 금지).
- 여러 모델로 시연했다면 모델별로 표를 반복 작성하십시오.
